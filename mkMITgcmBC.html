mkMITgcmBC

mkMITgcmBC [OPTIONS] COMMAND [ARGS]...

mds

Use input grid informations from grid info mds (XC,YC,RC,hFacC,…) files to generate MITgcm boundary conditions. The input file must be a CF-compliant NetCDF file and should contain a single data variable. If the input file contains multiple data variables, please use cdo operator “-selvar” to select a single data variable. Example:

  • mkMITgcmBC mds –grid-path mds_grid_info_files_directory_path/ –field T –boundary E –input “-mergetime [ input_data_*.nc ]”

  • mkMITgcmBC mds –grid-path mds_grid_info_files_directory_path/ –field U –boundary E –boundary N –input “-selvar,uvel input_data.nc”

mkMITgcmBC mds [OPTIONS]

Options

--input <input>

Required Input can be:

  1. A CF-compliant NetCDF file.

  2. A valid cdo option which will generate a NetCDF file.

e.g. “-mergetime input1.nc input2.nc input3.nc”

--field <field>

Required Boundary field name, i.e. T, S, U, V,…

e.g.; This will be used to generate files <field>_E.bin, <field>_W.bin,..

For field “U” and “V”, West and South grid coordinates of Arakawa-C will be used respectively.

For all other fields Center grid coordinates of Arakawa-C will be used.

--grid-path <grid_path>

Directory path where grid info mds files are

Default:

PosixPath('.')

--boundary <boundary>

boundary; can be defined multiple times

e.g. –boundary S –boundary N –boundary E

Default:

Options:

S | N | E | W

--addc <addc>

Add a constant to the input field

Default:

0.0

--mulc <mulc>

Multiply a constant to the input field

Default:

1.0
